자리잡다 – 추천 지도 UI (Frontend Demo)
무엇을 보여주나요?

지도 + 바텀시트 카드 조합으로, 사용자의 조건을 기반으로 선정된 추천 3곳을 한 화면에서 보여줍니다.

마커를 누르면 해당 장소 카드가 활성화되고, 카드를 스와이프하면 지도도 함께 이동합니다.

핵심 화면 구성

상단 앱바

현재 검색/추천 기준이 되는 건물·단지명 표시 (예: 메세나 폴리스).

검색 아이콘(확장 여지), 닫기 아이콘(뒤로 가기).

지도 영역 (Kakao Maps)

하늘색 마커: 현재 선택된 추천 지점(하이라이트).

회색 마커: 나머지 후보 지점.

(선택) 마커 내부에 **층 배지(B1/1F/2F)**를 넣어 층별 구분을 직관화.

바텀시트 카드

주소 라벨: 예) 서울 마포구 양화로 45

장소명/층 정보: 메세나 폴리스 지하 1층

핵심 정보: 임대료 추정(약 250만원/월), 주요 업종(디저트·카페)

하트(♥): 즐겨찾기/저장 표현(데모에선 로컬 상태로 처리).

푸터 CTA: “자세히 보기 / 위로 올리기” 등 상세 페이지나 카드 확장 동작.

인터랙션 시나리오

마커 클릭 → 카드 싱크
마커를 누르면 해당 후보가 하늘색 마커로 바뀌고, 바텀시트가 그 카드로 스크롤/활성화됩니다.

카드 스와이프 → 지도 싱크
좌우 스와이프로 카드 전환 시 지도 중심이 해당 후보 위치로 이동합니다.

층 선택(옵션)
좌측 세로 칩(B2/B1/1F/2F/ALL)에서 층을 선택하면 해당 층의 후보만 지도/카드에 노출됩니다.

추천 사유(Reason) 노출 (권장)

카드 상단 또는 중간에 한 줄로 노출:
예) 추천 이유: 합정역 도보 3분 · 예산 내 · 디저트 상권 적합

심사위원 관점에서 “왜 이 지점을 골랐는가”를 즉시 납득할 수 있어 신뢰감이 올라갑니다.

데이터 흐름(프론트 단독 데모)

이번 데모는 프론트만 동작하며 mock 데이터로 렌더링합니다.

이후 실제 백엔드 연동 시, 아래 계약으로 교체만 하면 됩니다.

기술 포인트

지도: Kakao Maps JS SDK

상태: React state(또는 Zustand)로 activePlaceId, activeFloor, savedIds 관리

동기화: 마커 클릭 ↔ 카드 활성화 양방향 연결

반응형: 모바일 기준 최적화(바텀시트 높이 35%~80% 가변)

심사 포인트(해커톤 데모용 멘트)

“추천 정확도와 UI 완성도에 집중했습니다.
추천 3건을 한눈에 비교하고, 클릭 순간 이유가 보이는 구조로 의사결정 시간을 줄입니다.”

“지도–카드 양방향 싱크와 층 필터링으로 복잡한 상권에서도 빠르게 후보를 좁힙니다.”

“백엔드 없이도 돌아가지만, 위 API 계약에 맞춰 바로 연동 가능하도록 설계했습니다.”
